# Object Oriented Design


### Abstract Data type
- Data type that is defined by programmer and not built into the language.

### Process 
1. Eliciting Requirements
    - Requirements are the needs or desires of the customer that must be fulfilled by the software system.
    - **User Story** : Simple, natural language description of a software requirement
        - As a [ role ], I want to [ goal ] so that [ reason].
2. Produce conceptual design (mock ups)
3. Produce technical design 

### Category of Objects
- Entity Objects : Entity objects represent real-world concepts or persistent business data.
    - Example : Customer, User, Account
- Boundary Objects : Boundary objects handle interaction between the system and external actors
    - Example : LoginPage, OrderScreen
- Control Objects : Control objects manage use-case logic and coordination.
    - Example : LoginController, OrderController
