# Scaling in Databases

### Realtional DB Scaling

- **SHarding** -
- **Read Replicas**: Create copies of the database to handle read requests, reducing the load on the primary database.
- **Partitioning**: Split large tables into smaller, more manageable pieces, improving performance and maintenance.
- **Caching**: Implement caching strategies to store frequently accessed data in memory, reducing database load.
- **Connection Pooling**: Use connection pools to manage database connections efficiently, minimizing overhead.
